### 场景 2：网络诊断（角色专业化）

#### 2.1 查询网关地址（失败案例）

**任务提交：**
```bash
python3 linux_agent_multi_client.py task submit \
  "查询本机所在的交换机地址或网关地址" \
  --role network --auto
```

**执行结果：**
```
任务 ID: 7ba2d516-ebfd-4a7b-bb14-9261ad316123
Agent: agent-003 (网络专家-01)
状态: failed
命令: ip route show default | grep -oP 'via \K\S+'

错误:
/bin/sh: ip: command not found
grep: invalid option -- P
```

**问题分析：** Agent 生成了 Linux 命令，但在 macOS 上不可用。

---

#### 2.2 查询网关地址（成功案例）

**任务提交（明确指定 macOS）：**
```bash
python3 linux_agent_multi_client.py task submit \
  "在 macOS 系统上查询默认网关地址" \
  --role network --auto
```

**执行结果：**
```
任务 ID: d053dbff-55f4-47ce-8bc7-4f7c94c46055
Agent: agent-003 (网络专家-01)
状态: success
命令: netstat -nr | grep default

输出:
default            192.168.1.1        UGScg                 en0
default            fe80::1%en0        UGcg                  en0
default            fe80::%utun0       UGcIg               utun0
default            fe80::%utun1       UGcIg               utun1
default            fe80::%utun2       UGcIg               utun2
default            fe80::%utun3       UGcIg               utun3
```

**关键信息：**
- **IPv4 网关：** 192.168.1.1
- **网络接口：** en0
- **IPv6 网关：** fe80::1%en0

**经验教训：** 在任务描述中明确指定操作系统类型，可以提高命令生成的准确性。


```
default            fe80::%utun1       UGcIg               utun1
```

**AI 分析亮点：**
1. ✅ **专业解读** - 解释了 utun 接口的作用
2. 🔍 **异常检测** - 确认无异常情况

## 💡 最佳实践

### 1. 明确指定操作系统

❌ **不推荐：**
```bash
"查询网关地址"
```

✅ **推荐：**
```bash
"在 macOS 上查询网关地址"
"在 Linux 上查询网关地址"
```

### 2. 使用角色专业化

```bash
# 网络相关任务使用网络专家
--role network

# 监控相关任务使用监控专家
--role monitor

# 安全相关任务使用安全专家
--role security
```

### 3. 利用 AI 分析

执行任务后，务必查看 AI 分析：
```bash
python3 linux_agent_multi_client.py task get <task_id>
```

AI 分析会提供：
- 结果验证
- 问题识别
- 解决方案
- 关键信息

### 4. 使用 --auto 标志

对于安全的查询命令，使用 `--auto` 跳过确认：
```bash
python3 linux_agent_multi_client.py task submit \
  "查看磁盘使用情况" --auto
```
