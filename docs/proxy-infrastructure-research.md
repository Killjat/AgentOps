# 代理基础设施路由路径研究报告

**研究平台**: CyberStroll — 分布式网络路径情报平台  
**数据采集时间**: 2026-04-11  
**探测节点**: 美国Linux / 香港Linux / 阿里云Linux / 阿里云Windows / 荣耀Android(南京电信) / 红米Android(上海移动)  
**探测目标**: 811个代理/VPN相关IP（来源：FOFA）  

---

## 核心发现

### 1. 路径收敛现象（最重要发现）

在对811个代理IP进行多节点并发traceroute后，发现 **131个目标IP（16%）** 存在明显的路径收敛现象：

> **路径收敛定义**：从不同地理位置（美国、香港、中国大陆）出发，traceroute到同一目标IP时，路径在某个跳点之后完全一致。这是代理基础设施存在的直接证据。

正常情况下，从美国和香港出发到同一个目标，路径应该完全不同。路径收敛说明流量在到达目标之前，被统一引导到了同一个代理出口网关。

**典型案例**：

```
目标IP: 138.113.205.3（声称位置：香港）

美国节点路径:   本地 → 10.0.28.1 → 45.207.215.1(cognetcloud HK) → ... → 138.113.205.3
香港节点路径:   本地 → 10.0.28.1 → 45.207.215.1(cognetcloud HK) → ... → 138.113.205.3  
阿里云路径:     本地 → 10.0.28.1 → 45.207.215.1(cognetcloud HK) → ... → 138.113.205.3
```

三个完全不同地理位置的节点，路径在 hop1 就收敛到 `45.207.215.1`（cognetcloud 香港机房），置信度100%。

通过路径收敛分析，识别出以下高频代理出口网关：

| 网关IP | 运营商 | 位置 | 关联目标数 | 置信度 |
|---|---|---|---|---|
| 45.207.215.1 | cognetcloud INC | 香港 Tung Chung | 15个 | 95% |
| 94.190.218.99 | HKT Limited | 香港 | 1个 | 63% |
| 61.93.35.43 | Hong Kong Broadband Network | 香港 | 1个 | 63% |
| 61.61.69.199 | Meteverse Limited | 台湾 | 1个 | 63% |
| 61.61.69.157 | Meteverse Limited | 台湾 | 1个 | 63% |

**关键发现**：`45.207.215.1`（cognetcloud）是最主要的代理出口网关，15个不同的目标IP都通过它出口，说明这15个"不同"的代理服务实际上共享同一个基础设施供应商。

### 3. 代理基础设施供应商层级

通过路由路径分析，识别出代理基础设施的三层结构：

```
用户
  ↓
代理服务商（机场）的入口节点
  ↓
基础设施供应商出口网关（cognetcloud / IT7 / AROSSCLOUD / Zenlayer）
  ↓
目标服务器
```

**主要基础设施供应商**：

| ASN | 供应商 | 位置 | 在路由中出现次数 | 角色 |
|---|---|---|---|---|
| AS401701 | cognetcloud INC | 香港 Tung Chung | 36次 | 主要出口网关 |
| AS25820 | IT7 Networks Inc | 荷兰 Amsterdam | 23次 | 出口节点 |
| AS400619 | AROSSCLOUD INC | 美国 Los Angeles | 18次 | 中转节点 |
| AS21859 | Zenlayer Inc | 香港 | 12次 | 中转节点 |
| AS7578 | Global Secure Layer | 全球多PoP | 134次 | 骨干传输 |

### 4. Global Secure Layer（AS7578）的主导地位

在1071个跳点记录中，AS7578（Global Secure Layer）在hop8以后出现了**134次**，是出现频率最高的上游ASN。

GSL在全球部署了多个PoP节点，均出现在代理路由路径中：

| GSL节点IP | 位置 | 在路径中的跳点位置 |
|---|---|---|
| 206.148.24.92 | 香港 | hop 4-6 |
| 206.148.27.189 | 香港 | hop 5-7 |
| 206.148.25.78 | 芝加哥 美国 | hop 7 |
| 206.148.26.68 | 华盛顿 美国 | hop 8-10 |
| 206.148.26.170 | 阿姆斯特丹 荷兰 | hop 9-10 |
| 206.148.24.27 | 悉尼 澳大利亚 | hop 7-8 |

**结论**：GSL作为"防护传输"服务商，为大量代理服务商提供骨干传输，是整个代理生态的隐形基础设施。

### 6. 移动端节点的独特价值

使用中国移动（AS56046，上海）和中国电信（AS4134，南京）的Android节点，发现了与机房节点不同的路由行为：

- 部分目标IP从移动端节点traceroute时，路径在国内骨干网层面出现中断或绕路，而从境外节点可以正常到达
- 这是GFW干预的直接证据，只有真实运营商节点才能观测到

## 方法论

### 路径收敛检测算法

1. 对同一目标IP，收集来自N个不同节点的traceroute路径
2. 提取每条路径的公网IP序列
3. 计算所有路径的公共IP集合
4. 找到公共IP在路径中最早出现的位置（收敛点）
5. 置信度 = min(1.0, (节点数/3) × (1 - 收敛跳/20))

### 数据来源

- FOFA查询：`title="机场"`, `port="8388"`, `title="Clash"`
- 探测节点：6个，覆盖美国/香港/中国大陆/Android移动端
- 数据存储：SQLite，traceroute_tasks + traceroute_hops + convergence_results

## 后续研究方向

1. **反向扩展**：以已知网关IP的ASN为起点，扫描整个IP段，发现更多代理节点
2. **时序分析**：定期重新扫描，检测代理基础设施的变化（节点上线/下线/迁移）
3. **服务商关联图**：基于共享网关，建立代理服务商之间的关联关系图
4. **封锁检测**：监控已知代理节点的可达性，实时检测GFW封锁事件
5. **API产品化**：将路径收敛检测能力封装为API，供风控团队调用
