# CyberAgentOps USB Agent 使用说明

## 这是什么

将此 U 盘插入你自己的电脑，运行启动脚本后，该电脑会自动作为一个 Agent 节点接入 CyberAgentOps 控制系统，你可以在控制台远程执行命令、查看系统状态、纳入 Swarm 多机任务。

---

## 快速开始

### Windows

1. 插入 U 盘，打开文件管理器找到 U 盘目录
2. 双击运行 `launch.bat`
3. 看到 "Agent 已在后台启动" 即成功

### macOS

1. 插入 U 盘，打开终端
2. 进入 U 盘目录：
   ```bash
   cd /Volumes/你的U盘名称
   ```
3. 运行启动脚本：
   ```bash
   bash launch.sh
   ```
4. 看到 "✅ Agent 已在后台启动" 即成功

### Linux

```bash
cd /media/你的用户名/U盘名称
bash launch.sh
```

---

## 连接后能做什么

登录控制台后，该机器会出现在 Agent 列表中，你可以：

- **单机任务**：直接向该 Agent 发送 shell 命令并查看输出
- **Swarm 任务**：将多台机器组合，用自然语言描述目标，AI 自动规划并分发任务
- **系统监控**：查看 CPU、内存、磁盘、网络实时状态
- **应用发现**：自动扫描运行中的服务和端口

## 查看运行日志

Agent 启动后日志写入 U 盘根目录的 `agent.log`：

```bash
# macOS / Linux
tail -f agent.log

# Windows（PowerShell）
Get-Content agent.log -Wait
```

---

## 停止 Agent

```bash
# macOS / Linux - 找到进程并终止
pkill -f cyberagent

# Windows（PowerShell）
Stop-Process -Name cyberagent-windows -Force
```

## 常见问题

**Q: macOS 提示"无法打开，因为无法验证开发者"**

打开「系统设置 → 隐私与安全性」，点击「仍要打开」，或运行：
```bash
xattr -d com.apple.quarantine bin/cyberagent-mac
```

**Q: 启动后控制台看不到这台机器**

检查网络是否能访问控制服务器，查看 `agent.log` 中的错误信息。

**Q: Agent 重启后还在吗**

不在，当前为手动启动模式。重启后需重新运行启动脚本。如需开机自启，联系管理员配置。
